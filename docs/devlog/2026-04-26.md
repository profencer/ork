# 2026-04-26 — ADR-0013 generic gateways land; demo kitchen-sink extends to `/api/gateways/*`

> Session focus: close out [`docs/adrs/0013-generic-gateway-abstraction.md`](../adrs/0013-generic-gateway-abstraction.md) (status **Implemented**), wire
> `ork-gateways` into `ork-api`, and make the public demo path prove the feature without
> requiring a second binary.

## Highlights

- New crate [`crates/ork-gateways/`](../../crates/ork-gateways) hosts HTTP/Kafka/MCP
  *adapters* only; `ork-core` keeps protocol-neutral
  [`Gateway`](../../crates/ork-core/src/ports/gateway.rs) + `GenericGatewayAdapter`
  per hex rules. Built-ins: `rest`, `webhook`, `event_mesh`, `mcp` (streamable HTTP,
  one MCP tool per local agent, `agent__<id>` naming).
- `ork-api` loads `[[gateways]]` at boot, merges public `Router<()>` routes (per-route
  `Extension` state so `Router::merge` works), spawns
  [`GatewayDiscoveryPublisher`](../../crates/ork-eventing/src/gateway_discovery.rs)
  per instance after a successful `Gateway::start`, and cancels shared tokens before
  per-gateway `shutdown` on process exit. Legacy
  `POST /api/webhooks/pipeline` delegates to
  [`run_pipeline_webhook`](../../crates/ork-gateways/src/webhook.rs) shared with
  the gateway adapter; pipeline no-ops now `warn!` on tenant/definition failures while
  keeping **202** fire-and-forget semantics (documented in the ADR reviewer block).
- **Demo:** [`demo/config/default.toml`](../../demo/config/default.toml) adds
  `demo-rest` and `demo-webhook`; [`demo/scripts/stage-2-cards.sh`](../../demo/scripts/stage-2-cards.sh)
  checks **401** without `X-Tenant-Id`, optional **200** REST round-trip when
  `MINIMAX_API_KEY` is set, and **202** on the unsigned webhook path. In-memory Kafka
  means we did not add an `event_mesh` exercise here — called out in
  [`demo/README.md`](../../demo/README.md).

## ADR-0016 artifact storage — implementation, gateway wiring, accept

### Highlights

- **Artifact stack:** `ArtifactStore` + `ArtifactMetaRepo` in [`crates/ork-core/src/ports/`](../../crates/ork-core/src/ports/); `ork-storage` (FS, chained router, S3 behind `--features s3` + MinIO smoke); [`migrations/007_artifacts.sql`](../../migrations/007_artifacts.sql) and [`crates/ork-persistence/`](../../crates/ork-persistence/); seven `artifact_*` tools and embed handlers; `GET /api/artifacts/...` proxy; inbound/outbound `Part::File` URI rewrite for A2A paths.
- **Gateways + artifacts:** [`GatewayDeps`](../../crates/ork-core/src/ports/gateway.rs) now carries `artifact_store` and `artifact_public_base` (mirroring primary A2A boot). **All four** built-in gateway adapters — [`rest`](../../crates/ork-gateways/src/rest.rs), [`webhook`](../../crates/ork-gateways/src/webhook.rs) (agent mode), [`mcp_gw`](../../crates/ork-gateways/src/mcp_gw.rs), [`event_mesh`](../../crates/ork-gateways/src/event_mesh.rs) — thread those into `AgentContext` so tool spill and artifact tools work off the main A2A route, not only `/a2a`.
- **Reviewer-driven fixes:** S3 [`list_objects_v2`](../../crates/ork-storage/src/s3.rs) loops continuation tokens (AWS caps at 1000 keys per call; SDK `is_truncated` is `Option<bool>` in our version). [`eligible_for_sweep`](../../crates/ork-persistence/src/postgres/artifact_meta_repo.rs) `JOIN tenants` and `COALESCE` `settings->>'artifact_retention_days'` with the global default for the **age** branch; task-completed branch still uses the global `task_days` parameter.
- **Process:** ADR-0016 status **Accepted**; [`docs/adrs/README.md`](../adrs/README.md) index; [`docs/adrs/metrics.csv`](../adrs/metrics.csv) row for 0016.
